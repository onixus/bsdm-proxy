# BSDM-Proxy: Конкурентный анализ и Roadmap развития

Этот документ представляет собой анализ проекта `bsdm-proxy` в контексте отчетов **Gartner Magic Quadrant for Security Service Edge (SSE)** и **Secure Access Service Edge (SASE)**, а также определяет стратегический Roadmap для достижения технологического и маркетингового превосходства.

---

## 1. Сравнение с лидерами рынка (Gartner SSE/SASE)

**Лидеры рынка по Gartner:** Zscaler (ZIA), Palo Alto Networks (Prisma Access), Netskope, Cloudflare.

| Возможность (Gartner Core SSE) | Лидеры рынка | BSDM-Proxy (Текущее состояние) | Оценка |
|--------------------------------|--------------|--------------------------------|--------|
| **Secure Web Gateway (SWG)** | Облачный масштаб, категоризация, SSL-инспекция | Нативный Rust, MITM, кэширование, ACL, фильтрация | 🟩 **Наравне / Превосходит (по скорости)** |
| **Advanced Threat Protection** | Sandbox, AI-based эвристика | ML-пайплайн в ClickHouse (C&C, Phishing, UEBA, Flight Risk) | 🟩 **Сильно**, но оффлайн (асинхронно) |
| **Cloud Access Security Broker (CASB)**| Контроль SaaS-приложений, API-интеграции | Частично (категоризация) | 🟨 **Отставание** (нет гранулярного контроля SaaS) |
| **Data Loss Prevention (DLP)** | OCR, Exact Data Match, Regex в реальном времени | Инспекция ограничена ICAP/Wasm | 🟥 **Отставание** (нет встроенного DLP-ядра) |
| **Zero Trust Network Access (ZTNA)**| Доступ к приватным приложениям без VPN | Нет (только Forward Proxy) | 🟥 **Отставание** |
| **Remote Browser Isolation (RBI)**| Пиксельный стриминг опасных сайтов | Нет | 🟥 **Отставание** |
| **Endpoint Agent** | Zscaler Client Connector, WARP | Управление через PAC-файлы/GPO | 🟨 **Отставание** (зависимость от ОС) |

### Чего не хватает для абсолютного технологического преимущества?

1. **Встроенного DLP-модуля (Data Loss Prevention)**. Сканирование исходящего трафика (POST-запросов и файлов) на утечку персональных данных (PII), кредитных карт и исходного кода.
2. **Контроля использования GenAI (AI-CASB)**. Gartner выделяет контроль над ChatGPT, Claude и Copilot как топ-приоритет 2025 года. Нам нужен контроль того, что сотрудники отправляют в LLM.
3. **Endpoint Agent'а (Легкого клиента)**. Проксирует трафик на BSDM-Proxy независимо от того, где находится сотрудник (в офисе или в кафе), обеспечивая Always-On Security.
4. **ZTNA (Zero Trust Network Access)**. Возможность работать не только как "выход" в интернет, но и как "вход" во внутренние сервисы компании с жесткой аутентификацией.

## 2. Маркетинговые преимущества (Unique Selling Propositions)

Чтобы конкурировать с многомиллиардными корпорациями, BSDM-Proxy должен бить в их слабые места.

1. **Data Sovereignty (Полный суверенитет данных)**
   * *Проблема лидеров:* Zscaler и Netskope гонят весь трафик в свои облака. Корпорации боятся отдавать им ключи SSL и телеметрию.
   * *Наше преимущество:* 100% On-Premise или Private Cloud. Полный контроль над данными, логами и ML-моделями.
2. **"Стеклянная коробка" ML-аналитики (White-box AI)**
   * *Проблема лидеров:* Их ИИ — это черный ящик. Ошибки ложного срабатывания невозможно объяснить.
   * *Наше преимущество:* Открытая аналитика на базе ClickHouse. Клиент видит фичи, бейзлайны, может писать свои SQL-модели поверх трафика.
3. **Rust-Native производительность и Эффективность**
   * *Проблема лидеров:* Тяжелые legacy-стеки (часто C++ эпохи 2000-х или Java), требующие огромных ресурсов (особенно для SSL-дешифровки).
   * *Наше преимущество:* Экстремально низкие задержки, асинхронный I/O, безопасная работа с памятью и интеграция с Wasm.
4. **Бесшовная расширяемость (Wasm)**
   * *Наше преимущество:* Вендорам требуются месяцы на добавление кастомных парсеров. У нас клиент может написать плагин на Go/Rust, скомпилировать в Wasm и загрузить "на лету".

## 3. Roadmap технологического и продуктового прорыва (Next-Gen)

Дорожная карта перехода от "Open-Source SWG" к "Enterprise-Grade SSE Platform".

### Q3 2026: Удержание контроля над ИИ (GenAI CASB & DLP)
* **LLM Data Guard:** Интеграция Wasm-плагина для инспекции запросов к API OpenAI / Anthropic. Блокировка отправки исходного кода и PII.
* **Inline DLP-движок (v1):** Встроенный сканер по регулярным выражениям (DLP) для POST/PUT запросов с поддержкой стриминга (без буферизации больших файлов в память).

### Q4 2026: Гибридная безопасность (Endpoint & ZTNA Foundation)
* **BSDM Connect (Легкий клиент):** Простой агент для Windows/macOS/Linux, использующий WireGuard-туннель для принудительного завертывания веб-трафика в корпоративный BSDM-Proxy.
* **Reverse Proxy / Identity-Aware Proxy (IAP):** Добавление функционала терминирования внешнего трафика ко внутренним ресурсам (зародыш ZTNA) с поддержкой OIDC (Okta, Keycloak).

### Q1 2027: Распределенная Enterprise-Экосистема
* **Global Session State:** Использование Redis/KeyDB для распределенных сессий (`session_id`) и Rate Limiting между репликами в мульти-кластерной K8s среде.
* **Real-time Threat Sync:** P2P-обмен индикаторами компрометации (IoC) и ML-скорами между разными инстансами (по модели Gossip).

### Q2 2027: Wasm-Marketplace и Ecosystem
* **BSDM Plugin Hub:** Портал с открытым исходным кодом, где сообщество может обмениваться Wasm-плагинами (например, парсеры специфичных проприетарных протоколов поверх HTTP).
* **AI-категоризация Zero-Day доменов:** Использование локальных SLM (Small Language Models) на лету для классификации ранее не известных доменов.

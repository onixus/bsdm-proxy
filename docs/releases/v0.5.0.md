## Сборка

```bash
cargo build --release \
  -p bsdm-proxy --bin proxy \
  -p cache-indexer --bin cache-indexer \
  -p alert-worker --bin alert-worker

./scripts/build-package.sh
# → dist/bsdm-proxy-0.5.0-linux-<arch>.tar.gz
```

Пакет включает `bin/proxy`, `bin/cache-indexer`, `bin/alert-worker`.

## Установка

```bash
tar xzf dist/bsdm-proxy-0.5.0-linux-x86_64.tar.gz
cd bsdm-proxy-0.5.0-linux-x86_64
sudo ./install.sh --create-user --systemd
sudo systemctl start bsdm-proxy bsdm-cache-indexer
# optional SIEM alerts:
# sudo systemctl enable --now bsdm-alert-worker
```

## Docker Compose

```bash
# Full stack (ClickHouse + Grafana + Alertmanager)
docker compose up -d --build

# SIEM webhook path (alert-worker)
ALERT_WEBHOOK_URL=https://siem.example/hooks/bsdm \
  docker compose --profile alerts up -d --build alert-worker

# Lite (no Kafka/CH)
./scripts/gen-ca.sh
docker compose -f docker-compose.lite.yml up -d --build
```

| Endpoint | URL |
|----------|-----|
| Grafana | http://localhost:3000 (Alerting → folder **BSDM M4**) |
| Alertmanager | http://localhost:9093 |
| Prometheus alerts | http://localhost:9091/alerts |
| ClickHouse | http://localhost:8123 |

## Upgrade с 0.3.2

Breaking config changes: **нет**. Новые опциональные компоненты:

1. `alert-worker` + `ALERT_WEBHOOK_URL` (profile `alerts` / systemd `bsdm-alert-worker`)
2. Compose service **alertmanager** (always-on in full stack) — set `ALERT_WEBHOOK_URL` to forward
3. Shannon / beacon env knobs — see [alerting.md](../alerting.md)
4. Lite path: `docker-compose.lite.yml`, `EVENT_SINK_URL`, `INDEX_STORE`

```bash
git pull
docker compose up -d --build
# or rebuild release package / Helm tag 0.5.0
```

## Сборка

```bash
cargo build --release -p bsdm-proxy --bin proxy -p cache-indexer --bin cache-indexer

./scripts/build-package.sh
# → dist/bsdm-proxy-0.3.1-linux-<arch>.tar.gz
```

## Установка

```bash
tar xzf dist/bsdm-proxy-0.3.1-linux-x86_64.tar.gz
cd bsdm-proxy-0.3.1-linux-x86_64
sudo ./install.sh --create-user --systemd
sudo systemctl start bsdm-proxy bsdm-cache-indexer
```

## Миграция с 0.3.0

### OpenSearch → ClickHouse

1. Остановить старый стек: `docker compose down`
2. Обновить до v0.3.1 (`git pull`, пересобрать образы или скачать release package)
3. `docker compose up -d --build` — ClickHouse поднимется с `scripts/clickhouse/http_cache.sql`
4. Исторические данные OpenSearch **не** мигрируются автоматически — re-index из Kafka retention или cold export
5. Grafana: dashboard **BSDM HTTP Traffic (ClickHouse)** вместо OpenSearch Dashboards

Подробнее: [clickhouse-analytics.md](../analytics/clickhouse-retrosearch.md) · [ADR 0002](../adr/0002-clickhouse-analytics.md)

### Native package

1. `sudo systemctl stop bsdm-proxy bsdm-cache-indexer`
2. Установить пакет 0.3.1 через `install.sh`
3. Обновить `cache-indexer.env`: `CLICKHOUSE_URL`, убрать `OPENSEARCH_*`
4. `sudo systemctl start bsdm-cache-indexer bsdm-proxy`

## Новые / изменённые env (cache-indexer)

| Переменная | Default | Описание |
|-----------|---------|----------|
| `CLICKHOUSE_URL` | `http://clickhouse:8123` | HTTP interface |
| `CLICKHOUSE_DATABASE` | `bsdm` | База |
| `CLICKHOUSE_TABLE` | `http_cache` | Таблица |
| `SEARCH_API_ENABLED` | `true` | REST `/api/search` |
| `SEARCH_API_TOKEN` | — | Bearer auth (опц.) |

Удалены: `OPENSEARCH_URL`, `INDEXER_BACKEND`, dual-write переменные.

## Известные ограничения

- **Breaking:** OpenSearch backend удалён — только ClickHouse
- `auth-kerberos` → `kerberos_keytab` (AGPL-3.0) — см. [licensing.md](../ops-and-dev/licensing.md)
- Grafana в compose — AGPL-3.0
- Shallalist (deprecated; use UT1) — внешняя БД, сервис discontinued; лицензия на стороне администратора

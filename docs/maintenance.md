# Поддержка документации

## Источник истины

Каноническая документация хранится в этом репозитории:

- `README.md` — краткое введение и быстрый старт;
- `docs/project-status.md` — фактическая зрелость функций;
- `docs/getting-started/` — установка и пилот;
- `docs/architecture/` — устройство и capacity planning;
- `docs/features/` — функциональные модули;
- `docs/analytics/` — ClickHouse, alerting и ML;
- `docs/ops-and-dev/` — эксплуатация и разработка;
- `docs/roadmap.md` — только будущие планы.

GitHub Wiki является публикуемым зеркалом. Не исправляйте скопированные страницы
только в Wiki: изменение будет потеряно при следующей синхронизации.

## Требования к изменениям

Документация изменяется в том же PR, что и пользовательская функция.

Перед commit:

```bash
python3 scripts/check-doc-links.py
```

Если изменяется структура Wiki:

```bash
python3 scripts/sync-wiki.py /path/to/bsdm-proxy.wiki
```

## Версии и статусы

- Текущую версию берите из `proxy/Cargo.toml`.
- Исторические номера версий остаются только в `docs/releases/`.
- Метки «основной», «beta» и «experimental» определены в
  `docs/project-status.md`.
- Наличие модуля или UI-страницы не означает production-ready.
- Roadmap не является доказательством реализации.

## Ссылки

- Внутри `docs/` используйте относительные ссылки с учётом текущего каталога.
- На файлы вне `docs/` также используйте корректный относительный путь.
- В Wiki ссылки преобразуются автоматически.
- Не добавляйте ссылки на старые имена файлов ради совместимости: вместо этого
  обновляйте все места использования.

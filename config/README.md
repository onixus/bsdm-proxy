# Configuration Directory

This directory stores configuration files used by BSDM-Proxy containers.

## Files

- `.env` - Environment variables (generated by web-ui)
- `docker-compose.yml` - Docker Compose config (generated by web-ui)
- `acl-rules.json` - Access Control List rules
- `custom-categories.json` - Custom URL categorization database
- `*.backup` - Automatic backups of changed configs

## Usage

### Via Web UI (Recommended)

1. Open http://localhost:8080
2. Configure settings in web interface
3. Click "Generate Configuration"
4. Click "Save & Apply"
5. Configs saved here automatically

### Manual Configuration

```bash
# Copy examples
cp .env.example .env
cp acl-rules.example.json acl-rules.json

# Edit configs
vim .env
vim acl-rules.json

# Restart services
docker-compose restart proxy
```

## Volume Mount

This directory is mounted to:
- `web-ui:/config` - For reading/writing configs
- `proxy:/app/.env` - For reading environment

## Security

**⚠️ Important:**
- `.env` contains sensitive data (passwords, tokens)
- Add this directory to `.gitignore`
- Do not commit actual configs to git
- Only commit `.example` files

## Backup

Configs are automatically backed up on change:
```bash
ls -la *.backup
```

Restore from backup:
```bash
cp .env.backup .env
docker-compose restart proxy
```
